# New Project Bootstrap

Use this checklist when starting a new project.

## 1. Create The Base Files

Recommended files:

- `project_instruction.md`
- `learning_assets.md`
- `README.md`
- `.gitignore`

Optional files:

- `concepts.md`
- `review_questions.md`
- `learning_log.md`
- `source_index.md`
- `decision_log.md`
- `templates/`

## 2. Add Common Instructions

Start from `common_project_instructions.md`, then add project-specific sections:

- project topic and goal;
- source priority;
- preferred answer language;
- expected output formats;
- recurring workflows;
- backup and privacy requirements.

## 3. Add Sources

Record project sources in `source_index.md` or `project_instruction.md`.

For each source, include:

- filename or URL;
- what it is for;
- whether it is a primary source;
- any important sections, page ranges, or keywords.

## 4. Initialize Git

Recommended flow:

1. Initialize or confirm git repository.
2. Add `.gitignore`.
3. Make the first commit.
4. Push to a private remote if the project contains personal notes or sensitive material.

## 5. Start The Learning Loop

For learning projects:

1. Study one unit.
2. Extract core structure and counterintuitive points.
3. Ask active-recall questions.
4. Update `learning_assets.md`.
5. Add durable concepts, review questions, or source indexes only when they will help future retrieval.

## 6. Instruction Update Rule

If a new recurring preference appears, propose an update before editing long-lived instruction files.
